What happens when you initialize a repository? Why do you need to do it?

When you initialize a repository it creates a repository for the files in a directory. It needs to be initialized in order to amalgamate all of the wanted files into the repository.



How is the staging area different from the working directory and the repository? What value do you think it offers?

The staging area is an intermediary for files before they are moved into the working directory and repository. It allows multiple files to be committed at the same time.



How can you use the staging area to make sure you have one commit per logical change?

The staging area can act as an intermediary between the working directory and the repository allowing a user to focus on the exact change needed.



What are some situations when branches would be helpful in keeping your history organized? How would branches help?
It would be helpful in large fundamental changes from the original file such as language change, change in project requirements, or refactoring of code. The branches would allow for a user to separate the changes and provide a clear direction of the history of a file.
